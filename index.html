5lb API

Документация для разработчиков · Версия 1.0
Базовый URL всех запросов
Base URL https://app.5lb.pro

API работает по протоколу HTTPS. Данные возвращаются в формате JSON.
Кодировка: UTF-8. Все строки с датами — формат ISO 8601.

Товар по артикулу 1С
GET

Статистика каталога

🔐 Авторизация

🔑 Bearer Token

Эндпоинт синхронизации защищён токеном. Токен необходимо передавать в заголовке каждого запроса:

Authorization: Bearer YOUR_SECRET_TOKEN
⚠️
Токен выдаётся администратором 5lb. Не передавайте токен третьим лицам и не включайте его в публичный код. Без токена сервер возвращает 401 Unauthorized.

Публичные GET-эндпоинты (список товаров, категории, бренды и т.д.) не требуют авторизации.

📦 Синхронизация товара из 1С

Главный эндпоинт для интеграции. При вызове товар создаётся, если его нет в базе, или обновляется, если уже существует — поиск ведётся по полю articleMain.

POST /api/catalog-5lb/sync Upsert товара по артикулу

Заголовки запроса

Заголовок Значение Обязательный
Authorization Bearer YOUR_TOKEN ДА
Content-Type application/json ДА

Тело запроса (JSON)

Поле Тип Обязательное Описание
articleMain string ДА Основной артикул — ключ для поиска в базе
productName string ДА Название товара
articleAdditional string|null нет Дополнительный артикул (вариация)
brand string|null нет Бренд
section1 string|null нет Категория 1 уровня
section2 string|null нет Категория 2 уровня
section3 string|null нет Категория 3 уровня
section4 string|null нет Категория 4 уровня
flavor string|null нет Вкус
color string|null нет Цвет
weightVolume string|null нет Вес / объём: "900 г", "1 л"
servings string|null нет Количество порций
quantityInPack string|null нет Кол-во в упаковке
country string|null нет Страна производителя
releaseForm string|null нет "Порошок", "Капсулы", "Таблетки"…
description string|null нет Описание товара
howToUse string|null нет Способ применения
ingredients string|null нет Состав / ингредиенты
compositionPerServing string|null нет Состав на порцию (КБЖУ)
ℹ️
Поле imageUrl не передаётся — фотографии загружаются отдельно через CRM 5lb. Необязательные поля можно опускать или передавать как null.

Ответ при успехе — 200 OK

JSON 
{
  "ok": true,
  "action": "updated",  // "created" или "updated"
  "item": {
    "id": "abc123",
    "articleMain": "1010002",
    "productName": "Whey Gold Standard",
    "imageUrl": "https://app.5lb.pro/images/catalog/1010002-CLYLW-VOL700.png",
    // ... остальные поля товара
  }
}

Коды ответа

200Товар успешно создан или обновлён
400Отсутствует обязательное поле (articleMain или productName)
401Токен отсутствует, неверный или истёк
500Внутренняя ошибка сервера

Ответ при ошибке

JSON 
{
  "ok": false,
  "error": "Описание ошибки"
}

🖥️ Пример интеграции на языке 1С

1C (BSL) 
// Соединение с сервером
Соединение = Новый HTTPСоединение(
    "app.5lb.pro", 443,
    , , ,
    30,
    Новый ЗащищенноеСоединениеOpenSSL()
);

// Заголовки
Заголовки = Новый Соответствие;
Заголовки["Content-Type"]  = "application/json";
Заголовки["Authorization"] = "Bearer ВСТАВЬТЕ_ВАШ_ТОКЕН";

// Данные товара
Данные = Новый Структура;
Данные.Вставить("articleMain",  Товар.Артикул);
Данные.Вставить("productName",  Товар.Наименование);
Данные.Вставить("brand",        Товар.Бренд);
Данные.Вставить("section1",     Товар.Раздел1);
Данные.Вставить("section2",     Товар.Раздел2);
Данные.Вставить("country",      Товар.СтранаПроисхождения);
Данные.Вставить("weightVolume", Товар.Вес);
Данные.Вставить("flavor",       Товар.Вкус);

// Сериализация в JSON
ЗаписьJSON = Новый ЗаписьJSON;
ЗаписьJSON.УстановитьСтроку();
ЗаписатьJSON(ЗаписьJSON, Данные);
ТелоJSON = ЗаписьJSON.Закрыть();

// Отправка запроса
Запрос = Новый HTTPЗапрос("/api/catalog-5lb/sync", Заголовки);
Запрос.УстановитьТелоИзСтроки(ТелоJSON, КодировкаТекста.UTF8);
Ответ = Соединение.ВызватьHTTPМетод("POST", Запрос);

// Обработка ответа
Если Ответ.КодСостояния = 200 Тогда
    ЧтениеJSON = Новый ЧтениеJSON;
    ЧтениеJSON.УстановитьСтроку(Ответ.ПолучитьТелоКакСтроку());
    Результат = ПрочитатьJSON(ЧтениеJSON);
    // Результат.action = "created" или "updated"
    Сообщить("Товар " + Результат.action);
Иначе
    Сообщить("Ошибка: " + Ответ.КодСостояния);
КонецЕсли;
⚠️
Если возникает ошибка "Узел не прошёл проверку" (SSL) — убедитесь, что используется ЗащищенноеСоединениеOpenSSL() с актуальными корневыми сертификатами Let's Encrypt.

GET /api/catalog-5lb Список товаров с пагинацией и фильтрами

Параметры запроса

Параметр Тип По умолчанию Описание
page number 1 Номер страницы
limit number 20 (макс. 100) Товаров на странице

brand string — Фильтр по бренду

🖼️ Изображения товаров

Изображения хранятся на сервере и доступны публично без авторизации.

Формат URL

Если есть доп. артикул:
https://app.5lb.pro/images/catalog/articleMain-articleAdditional.png
Если доп. артикул пустой:
https://app.5lb.pro/images/catalog/articleMain.png

Пример

URL
https://app.5lb.pro/images/catalog/1010002-CLYLW-VOL700.png
ℹ️
В ответе API поле imageUrl всегда содержит полный URL изображения или null, если фото не загружено.

⛔ Коды ошибок

Код Статус Причина
200 OK Запрос выполнен успешно
400 Bad Request Отсутствует обязательное поле или некорректные данные
401 Unauthorized Токен не передан, неверный или истёк
500 Server Error Внутренняя ошибка на стороне сервера

Формат ошибки

JSON
{
  "ok": false,
  "error": "Описание ошибки"
}
